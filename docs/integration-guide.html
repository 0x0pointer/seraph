Integration patterns

Seraph supports three integration patterns — from the simplest one-line wrapper to a fully transparent proxy that requires zero changes to your LLM client.

Pattern 1
Scan API (direct)
Call /api/scan/prompt and /api/scan/output from your application code. Full control, works with any LLM.
POST /api/scan/prompt
Pattern 2
Universal Hook
Single endpoint for gateways (Nginx, Traefik, Envoy, AWS API GW). Any direction, any platform.
POST /api/integrations/hook
Pattern 3
Transparent Proxy
OpenAI-compatible reverse proxy. Change one URL — no code changes. Scans both directions automatically.
POST /api/integrations/proxy
LiteLLM users: dedicated /api/integrations/litellm/pre_call and /api/integrations/litellm/post_call hooks are available — see the LiteLLM section.
Authentication

Connection keys vs JWT tokens

For production applications, use a connection key (ts_conn_…) rather than a user JWT. Connection keys never expire, are scoped to a named environment, and give each application its own independent metrics, spend tracking, and guardrail configuration.

Step 1

Create a connection in the dashboard

Go to Dashboard → APIs and click + New connection. Give it a descriptive name — for example production-chatbot or customer-portal-staging — and set the environment tag.

  • Copy the generated key immediately — it is only shown in full once.
  • Optionally set per-token pricing to track monthly spend.
  • Optionally set a monthly alert limit and hard spend cap.
  • Optionally set a violation-rate threshold to auto-block the key if abuse spikes.

Step 2

Add the key to your environment

Store the connection key in an environment variable — never hard-code it in source code. Treat it with the same care as a database password.

bash (.env) 
# .env — server-side only, never commit this file
SERAPH_API_URL=https://seraph.io          # base URL of your Seraph instance
SERAPH_CONNECTION_KEY=ts_conn_abc123...         # from Dashboard → APIs → copy key
OPENAI_API_KEY=sk-...                          # your LLM provider key
Add .env to your .gitignore. If a key is ever compromised, delete the connection in the dashboard and create a new one — past audit logs are retained.
Step 3 — Pattern 1

Integrate via the Scan API (direct)

The most flexible option. Call /api/scan/prompt before sending to your model and /api/scan/output before returning the response. Use the connection key as the Bearer token.

python
import os, httpx

API_URL = os.environ["SERAPH_API_URL"]
KEY   = os.environ["SERAPH_CONNECTION_KEY"]

headers = {
    "Authorization": f"Bearer {KEY}",
    "Content-Type": "application/json",
}

def chat(user_message: str) -> str:
    # 1 — Scan the incoming prompt
    scan = httpx.post(f"{API_URL}/api/scan/prompt",
                      json={"text": user_message},
                      headers=headers, timeout=30).json()

    if not scan["is_valid"]:
        return "Sorry, I can't help with that."  # log internally

    # 2 — Call LLM with sanitized_text (may have redactions applied)
    llm_response = call_your_llm(scan["sanitized_text"])

    # 3 — Scan the AI response
    out = httpx.post(f"{API_URL}/api/scan/output",
                     json={"text": llm_response, "prompt": user_message},
                     headers=headers, timeout=30).json()

    if not out["is_valid"]:
        return "Sorry, I can't provide that response."

    return out["sanitized_text"]
typescript
const API_URL = process.env.SERAPH_API_URL!;
const KEY   = process.env.SERAPH_CONNECTION_KEY!;

const seraph = async (path: string, body: object) => {
  const res = await fetch(`${API_URL}${path}`, {
    method: "POST",
    headers: { "Content-Type": "application/json", Authorization: `Bearer ${KEY}` },
    body: JSON.stringify(body),
  });
  if (!res.ok) throw new Error(`Seraph ${res.status}`);
  return res.json();
};

async function chat(userMessage: string): Promise<string> {
  const scan = await seraph("/api/scan/prompt", { text: userMessage });
  if (!scan.is_valid) return "Sorry, I can't help with that.";

  const llmResponse = await callYourLLM(scan.sanitized_text);

  const out = await seraph("/api/scan/output", {
    text: llmResponse, prompt: userMessage,
  });
  if (!out.is_valid) return "Sorry, I can't provide that response.";

  return out.sanitized_text;
}

  -H "Content-Type: application/json" \
  -d '{"text": "Seraph scans every prompt...", "prompt": "What are your security features?"}'
Step 4 — Pattern 2

Universal Hook

A single endpoint that works with any gateway or reverse proxy that can make an HTTP callback — Nginx auth_request, Traefik forwardAuth, Envoy ext_authz, AWS API Gateway, Apigee, Tyk, or custom middleware. No SDK, no dependencies. One POST in, one decision out.

Endpoint: POST /api/integrations/hook
Auth: Authorization: Bearer ts_conn_…
Returns: 200 → allowed  |  400 → blocked (gateway should reject)
python
import os, httpx

HOOK_URL = f"{os.environ['SERAPH_API_URL']}/api/integrations/hook"
headers  = {"Authorization": f"Bearer {os.environ['SERAPH_CONNECTION_KEY']}"}

def scan(text: str, direction: str = "input", prompt: str = "") -> dict:
    resp = httpx.post(
        HOOK_URL,
        json={"text": text, "direction": direction, "prompt": prompt},
        headers=headers, timeout=30,
    )
    if resp.status_code == 400:
        return {"allowed": False, "detail": resp.json().get("detail")}
    resp.raise_for_status()
    return {"allowed": True, **resp.json()}

# Scan input
result = scan(user_message, direction="input")
if not result["allowed"]:
    return {"error": result["detail"]}

# Scan output
result = scan(ai_response, direction="output", prompt=user_message)
if not result["allowed"]:
    return {"error": "Response blocked by guardrails."}
nginx.conf
# Nginx auth_request — every request is pre-screened by Seraph
location /api/ {
    auth_request /seraph-check;
    auth_request_set $skf_status $upstream_status;
    proxy_pass http://your-llm-service;
}

location = /seraph-check {
    internal;
    proxy_pass          https://seraph.io/api/integrations/hook;
    proxy_method        POST;
    proxy_set_header    Authorization "Bearer ts_conn_abc123...";
    proxy_set_header    Content-Type  "application/json";
    proxy_set_body      '{"text": "$request_body", "direction": "input"}';
    proxy_pass_request_body off;
}

Step 5 — Pattern 3

Transparent Proxy

An OpenAI-compatible reverse proxy. Change one URL in your client — that's it. Seraph scans every request and response automatically, then forwards the (sanitized) traffic to your upstream LLM. Works with the OpenAI SDK, LangChain, and any OpenAI-compatible client.

Endpoint: POST /api/integrations/proxy
Extra headers: X-Upstream-URL (your LLM base URL) · X-Upstream-Auth (your LLM key)
python
from openai import OpenAI
import os

# Before — direct to OpenAI
# client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

# After — route through Seraph proxy (zero other changes)
client = OpenAI(
    base_url=f"{os.environ['SERAPH_API_URL']}/api/integrations/proxy/v1",
    api_key=os.environ["SERAPH_CONNECTION_KEY"],
    default_headers={
        "X-Upstream-URL":  "https://api.openai.com",
        "X-Upstream-Auth": f"Bearer {os.environ['OPENAI_API_KEY']}",
    },
)

# All your existing code stays the same
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Hello!"}],
)
# Seraph has already scanned input + output transparently
typescript
import OpenAI from "openai";

const client = new OpenAI({
  baseURL: `${process.env.SERAPH_API_URL}/api/integrations/proxy/v1`,
  apiKey:  process.env.SERAPH_CONNECTION_KEY,
  defaultHeaders: {
    "X-Upstream-URL":  "https://api.openai.com",
    "X-Upstream-Auth": `Bearer ${process.env.OPENAI_API_KEY}`,
  },
});

// All existing code unchanged — scanning is fully transparent
const response = await client.chat.completions.create({
  model: "gpt-4o-mini",
  messages: [{ role: "user", content: "Hello!" }],
});
python (LangChain)
from langchain_openai import ChatOpenAI
import os

# Route LangChain through the Seraph transparent proxy
llm = ChatOpenAI(
    model="gpt-4o-mini",
    openai_api_base=f"{os.environ['SERAPH_API_URL']}/api/integrations/proxy/v1",
    openai_api_key=os.environ["SERAPH_CONNECTION_KEY"],
    model_kwargs={
        "extra_headers": {
            "X-Upstream-URL":  "https://api.openai.com",
            "X-Upstream-Auth": f"Bearer {os.environ['OPENAI_API_KEY']}",
        }
    },
)

# Every LangChain call is now scanned automatically
result = llm.invoke("What can you help me with?")
Step 6 — LiteLLM

LiteLLM custom guardrail

If you use LiteLLM as your LLM gateway, Seraph provides native pre_call, post_call, and during_call hook endpoints. Register them in your LiteLLM config and every call is scanned automatically.

yaml (litellm_config.yaml)
# litellm_config.yaml
model_list:
  - model_name: gpt-4o-mini
    litellm_params:
      model: openai/gpt-4o-mini
      api_key: "os.environ/OPENAI_API_KEY"

guardrails:
  - guardrail_name: seraph
    litellm_params:
      guardrail: custom
      mode: pre_call
      guardrail_endpoint: https://seraph.io/api/integrations/litellm/pre_call
      default_on: true
      api_key: "os.environ/SERAPH_CONNECTION_KEY"

  - guardrail_name: seraph-output
    litellm_params:
      guardrail: custom
      mode: post_call
      guardrail_endpoint: https://seraph.io/api/integrations/litellm/post_call
      default_on: true
      api_key: "os.environ/SERAPH_CONNECTION_KEY"
/litellm/pre_callinput scan
Called before forwarding the prompt to the LLM. Scans the last user message. Returns 200 (allowed) or 400 (blocked — LiteLLM surfaces this as an error).
/litellm/post_calloutput scan
Called after the LLM responds. Scans the assistant reply. Returns 200 or 400.
/litellm/during_callstreaming
Called mid-stream. Treated identically to pre_call — scans the last user message. Seraph does not scan partial stream chunks.

The core of chatbot/server.py:

python — chatbot/server.py (core logic)
SERAPH_KEY = os.getenv("SERAPH_CONNECTION_KEY", "")

@app.route("/chat", methods=["POST"])
def chat():
    user_message = request.json["message"]

    # ── 1. Scan user input ───────────────────────────────────────────
    input_scan = requests.post(
        f"{SERAPH_API_URL}/api/scan/prompt",
        json={"text": user_message},
        headers={"Authorization": f"Bearer {SERAPH_KEY}"},
        timeout=30,
    ).json()

    if not input_scan["is_valid"]:
        return jsonify({"blocked": True, "message": "Message blocked by guardrails."})

    safe_message = input_scan["sanitized_text"]   # use this, not the original

    # ── 2. Call OpenAI ───────────────────────────────────────────────
    completion = openai_client.chat.completions.create(
        model=OPENAI_MODEL,
        messages=[..., {"role": "user", "content": safe_message}],
    )
    ai_response = completion.choices[0].message.content

    # ── 3. Scan AI response ──────────────────────────────────────────
    output_scan = requests.post(
        f"{SERAPH_API_URL}/api/scan/output",
        json={"text": ai_response, "prompt": user_message},
        headers={"Authorization": f"Bearer {SERAPH_KEY}"},
        timeout=30,
    ).json()

    if not output_scan["is_valid"]:
        return jsonify({"blocked": True, "message": "Response blocked by guardrails."})

    return jsonify({"reply": output_scan["sanitized_text"]})

1 — Open the Guardrails panel

In Dashboard → APIs, find the connection card and click the Guardrails button (highlighted in purple when custom guardrails are active).

2 — Enable "Use custom guardrails"

When on, only the guardrails you select here run for this connection key. The global set is ignored for this key.

3 — Toggle scanners on or off

Use Input / Output tabs. Scanners marked globally off can still be enabled per-connection — they override the global setting.

4 — Adjust thresholds per scanner

For scanners with a numeric sensitivity (Toxicity, PromptInjection, BanTopics, etc.) you can override the threshold per connection without changing the global default.

5 — Test before saving

The Test scan panel runs the exact guardrail set configured for this connection — no ts_conn_ key needed. Type a prompt and click Run to verify.

Best practices

Production checklist

✓ Store the connection key securely

Keep it in an environment variable or secrets manager. Never commit it to your repository or expose it client-side.

✓ Always use sanitized_text

Pass sanitized_text to your model and return it to your user. The Secrets scanner silently redacts credentials — bypassing this leaks them to the model.

✓ Set a request timeout

Always set a timeout so a slow response never hangs your app. 30 seconds is a safe maximum.

python
httpx.post(url, json=body, headers=headers, timeout=10.0)

✓ Decide fail-open vs fail-closed

If Seraph is unreachable, decide whether to let the message through (fail open) or reject it (fail closed). The reference chatbot defaults to fail-open for availability.

✓ Pass actual token counts for spend tracking

Include input_tokens and output_tokens from your LLM's response for accurate monthly spend figures in the dashboard.

python
usage = completion.usage
httpx.post(f"{API_URL}/api/scan/output", json={
    "text": ai_response, "prompt": user_message,
    "input_tokens": usage.prompt_tokens,
    "output_tokens": usage.completion_tokens,
}, headers=headers)

✓ Log audit_log_id

Store it alongside your application logs to cross-reference scan decisions with user sessions in the Seraph audit dashboard.

✓ Self-hosting security

Set a strong SECRET_KEY in backend/.env, configure CORS_ORIGINS to your frontend domain, run behind a reverse proxy with TLS, and back up seraph.db regularly (or migrate to PostgreSQL).
